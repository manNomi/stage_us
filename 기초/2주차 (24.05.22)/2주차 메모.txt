CSS

* 정의 : html로 짜여진 틀을 꾸며주는 역할 
- style을 입혀주는 언어 
- stule이라고 부르자 
- 많이 아는게 중요한 것 (명령어 구조가 엄청 많음)
- "내가 힘들게 많든 것을 누군가는 쉽게 하고 있겠구나" 라고 생각해야함

* 동작 구조 
- html이 css를 찾아서 가져오는 구조 X 
- css가 적절한 html 태그에 가서 붙는 형태 
   -> css가 중심인 형태 

1. css만을 미리 먼저 작성해 둘 수 있다. 
	-> p태그 글씨는 파랑이다 
		-> 모든 p 태그는 파랑이 될것임
2. 한번 작성 해둔 것으 여러번 재활용 가능하다. 
	-> 태그에 이름을 붙일 수 있다. 
	-> box라는 이름을 가진 태그는 무조건 커진다 
		-> 재활용이 가능하다 

* 작성구조
- 디자인 패턴 : 문법이 아닌 양식이다 / 코드를 어떤 형태로 쓰겠느냐 

1. inline-style 
- 초창기에 쓰던 방법
- html 태그에 style 속성 붙여서 작성하는 방식 
- 절대 쓰면 안되는 패턴 
	-> 다른 개발자가 유지보수가 불가능 

2. <style>태그
- html 파일에 <head>태그 안쪽에 , <style>태그를 만들어서 css 문법 적는 방법 
- 거의 쓰지 않음
-> 특정 페이지가 아예 중복이 없을때 사용 할 수 도 있음
- 팝업창 정도 사용할때가 있음 

3. 파일 분할
- css파일을 html과 별개로 분리하는 방법 
- 가장 많이 사용하는 방법 

-> 페이지 별로 공통 되는 부분이 있는데 그 부분을 일일이 수정해야하기 때문에 파일분할로 사용한다 
-> EX) aside 나 footer 같은 경우 모든 페이지에서 사용하는데 스타일 태그 방식을 사용하면 유지보수가 어렵기때문
