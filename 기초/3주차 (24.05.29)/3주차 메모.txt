**JavScript

- html 역할 : 구조
- css 역할 : 그 구조 꾸미기
- js 역할 : 사용자의 행동에 따라서 화면에 변화를 줄떄 
ㄴ 이벤트 (사용자의 행동)
ㄴ js는 이벤트를 달아주는 언어 

- 시작 전 
ㄴ html과 css 같은 경우는 마크업 언어다 <- 프로그래밍 언어가 아님 
ㄴ js는 프로그래밍 언어이다 (수식 , 논리, 에러 &버그 존재)

- 매우 각광 받는 언어 
ㄴ 심화 단계 가면 js 100% 사용함 -> js 중요 

- 디자인 패턴
->코드를 쓰는양식
* oocs -> 클래스 쪼개기
* bem -> 아이디 

1.함수형 프로그래밍 방식
ㄴ함수라는것 위주로 쓰는것
->신흥강자

2.객체형 프로그래밍 방식
ㄴ객체라는걸 위주로 쓰는것 3
->예전부터 되어오던 것 

혹시 * class 방식이 나오면 보지 말것 <- 객체형 프로그래밍 방식이기에 무시하자 

**함수

- 정의
ㄴ x값을 넣으면 y값이 도출되는 수식
ㄴ 매개변수를 넣으면 리턴값이 도출되는 코드 다발 
ㄴ 매개변수나 리턴값은 여러개 일 수 도 있고 없을 수 도 있다 

-왜 ?
1. 중복코드 제거 
-> 필요할때마다 사용 (재사용 가능)

2. 가독성 증가
ㄴ긴코드를 일정 범위마다 끊어줄 수 있음

3. 에러와 버그 찾기의 용이성 (방화벽)
ㄴ일정구역 별로 나누 었기 때문에 특정 함수에서 오류가 나면 
   그 함수만 확인하면 된다
-함수를 쓰는 이유를 정리하자면 , 코드를 논리적으로 분할
